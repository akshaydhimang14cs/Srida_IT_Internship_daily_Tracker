# SRIDA IT Internship Daily Tracker

## Overview
This repository is created to maintain a structured record of my day-to-day internship progress at SRIDA IT. It documents the modules covered, daily learning activities, weekly progress summaries, and supporting evidence related to the internship work.

## Purpose
The purpose of this repository is to:

1. Track daily internship activities in an organized format.
2. Maintain module-wise learning records.
3. Summarize weekly progress for review.
4. Keep supporting references and work in one place.
5. Provide a professional and transparent progress record for review.

## Modules Covered
The internship currently includes the following modules:

1. Risk Assessment
2. Compliance Audit
3. Incident Response
4. GRC Simulation
5. Cybersecurity Audits
6. Mock Interviews

## Repository Structure
```text
shrida_it_internship_dialy_tracker/
```

## Folder Details

### daily-log
Contains daily markdown files with date-wise progress updates.  
Each file includes:
- Date
- Module name
- Chapter or topic covered
- Summary of what was learned
- Status of the day’s work

### weekly-summary
Contains weekly summaries of completed activities, progress made, and key learnings for that week.

### modules
Contains module-wise notes and summaries based on completed learning sections.

## Daily Update Method
This repository will be updated primarily through new files in the `daily-log` folder.
